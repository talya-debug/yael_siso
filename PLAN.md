# PLAN — מערכת ניהול משרד עיצוב פנים (yael_siso)

## טכנולוגיה
- **Frontend:** React + Vite + Tailwind CSS
- **Backend/DB:** Supabase (מסד נתונים + הרשאות + אחסון קבצים)
- **Deploy:** Vercel (חינמי, מחובר ל-GitHub)

## משתמשים והרשאות
- **Admin (2):** גישה לכל המודולים
- **User (8):** גישה מוגבלת לפי תפקיד

## מודולים (בסדר פיתוח)

### שלב 1 — תשתית
- [ ] הגדרת Supabase (DB + Auth)
- [ ] React app עם ניווט בסיסי
- [ ] כניסה / התנתקות + הרשאות

### שלב 2 — ליבת המערכת
- [ ] תכולות — קטלוג משימות לבחירה (לקוח בוחר מה הוא צריך)
- [ ] לקוחות — כרטיס לקוח
- [ ] פרויקט — נפתח אוטומטית עם אישור הצעה, כולל טיימליין משימות

### שלב 3 — פיצ'רים עסקיים
- [ ] גבייה מלקוחות — מעקב תשלומים
- [ ] גבייה מספקים — הזמנות רכש + תשלומים
- [ ] החתמת לקוח — חתימה דיגיטלית על מסמכים/משימות

### שלב 4 — תמיכה
- [ ] יומן עבודה — רישום שעות ופעילויות
- [ ] ריכוז ידע — ויקי פנימי / מסמכים

## מבנה DB (Supabase)
```
clients         — לקוחות
projects        — פרויקטים (שויך ללקוח)
tasks           — משימות (שויכות לפרויקט)
contents        — קטלוג תכולות
billing_clients — גבייה מלקוחות
billing_vendors — גבייה מספקים
work_log        — יומן עבודה
knowledge       — ריכוז ידע
users           — משתמשים + תפקידים (מנוהל ע"י Supabase Auth)
```
